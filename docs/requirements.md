# Requirements

This document describes what the Food Waste Tracker app should do over time.

## Initial Scope

The first working version should focus on the core food tracking workflow.

Users should be able to:

- add food items
- save an expiry date for each food item
- view all active food items
- see which foods expire soon
- mark food as consumed
- mark food as wasted
- manage a shopping list

## Later Versions

Later versions may add:

- food photo upload or attachment
- waste statistics
- charts or summaries
- PWA support
- offline-friendly behavior
- improved filtering and sorting

## Out of Scope for Now

These features are not part of the current documentation-only step:

- frontend implementation
- backend implementation
- database setup
- authentication
- image upload
- package installation
- deployment

## Learning Requirements

The project should be built step by step. Each step should stay small enough to understand before moving on to the next one.
